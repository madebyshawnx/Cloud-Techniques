# Hosting a Static Website on Azure (Blob Storage)

This lab demonstrates how to host a simple static website using Azure Storage and Blob Service.

## Watch me build this lab
https://www.loom.com/share/95c44fe345c64003a5e720a0336ec0ba

**What this video shows:** Creating an Azure Storage Account, enabling static website hosting, uploading HTML files, and accessing the live endpoint.

---

## Objective
Deploy and host a static HTML website using Azure Blob Storage without provisioning a traditional web server.

---

## Skills Practiced
- Creating and configuring Azure resources  
- Deploying and managing Azure Storage Accounts  
- Enabling static website hosting in Azure  
- Uploading files to Blob Storage  
- Working with storage endpoints and URLs  
- Troubleshooting web hosting errors  

---

## Tools Used
- Microsoft Azure Portal  
- Azure Storage Account  
- Azure Blob Storage  
- Static Website Hosting (Azure feature)  
- Web browser for endpoint testing  
- HTML file (example: `index.html`, `404.html`)  

---

## Steps Performed

1. **Created Storage Account**  
   Created a new Azure Storage Account within an existing resource group using default configuration settings and a nearby region.  
   (Optional screenshot) docs/step-1.png

2. **Reviewed Deployment**  
   Confirmed successful deployment of the storage resource before proceeding with configuration.  
   (Optional screenshot) docs/step-2.png

3. **Enabled Static Website Hosting**  
   Navigated to the Static Website settings under Data Management and enabled static hosting for the storage account.  
   (Optional screenshot) docs/step-3.png

4. **Configured Index and Error Documents**  
   Defined the default index document (example: `index.html`) and error document (example: `404.html`) for website routing.  
   (Optional screenshot) docs/step-4.png

5. **Copied Static Website Endpoint**  
   Identified and copied the primary endpoint URL generated by Azure for accessing the hosted site.  
   (Optional screenshot) docs/step-5.png

6. **Accessed Blob Container (Web)**  
   Navigated to the automatically created `$web` container within Blob Service to manage website files.  
   (Optional screenshot) docs/step-6.png

7. **Uploaded Website Files**  
   Uploaded the static HTML file (example: `index.html`) into the `$web` container to serve as the website content.  
   (Optional screenshot) docs/step-7.png

8. **Tested Website Endpoint**  
   Pasted the endpoint URL into a browser to validate website availability and troubleshoot initial content errors.  
   (Optional screenshot) docs/step-8.png

9. **Verified Website Functionality**  
   Confirmed that the static site rendered successfully from the Azure-hosted endpoint.  
   (Optional screenshot) docs/step-9.png

---

## Outcome
Successfully hosted and accessed a static HTML website using Azure Blob Storage without deploying a dedicated web server.
